Rules for Reporting Mac Linux USB Loader and Enterprise Bugs

Thank you for choosing to report a bug in Mac Linux USB Loader or Enterprise. In order to ensure bug reports are clear and understandable, and purvey enough information to allow me to fix the bug, I have established some ground rules for bug reports to ensure that your bug gets cleared in as quickly a matter possible and does not waste anyone's time. Reports that do not follow these rules will be closed:

  1. It is not a bug if the behavior is documented. To verify that the behavior is not documented, please check the Mac Linux USB Loader wiki and the Mac Linux USB Loader and Enterprise websites.
  2. Please make sure that the bug report has not already been filed. You can do that here and here. Be sure to check both open and closed issues. If a similar report has already been filed, do not file a new report. Instead, add any new information to the already existing report and, if it is closed, re-open it if necessary.
  3. Bug reports must be reproducable. You must demonstrate that the bug occurs every time under a specific set of circumstances. You should provide at least an account of exactly what you did leading up to the bug occurring (including switching to other programs, clicking on program notifications, opening a file, etc) but steps to reproduce are appreciated. If the bug reported results in a crash, please post in the report the Apple crash report.
  4. I need your basic information, like operating system version, version of Mac Linux USB Loader you are using (including if you are using a stable, pre-release or custom/self-compiled build), and how many times this bug has occurred for you before in your knowledge.

If the bug occurs with creating, configuring, or using Mac Linux USB Loader, file a bug report for Mac Linux USB Loader. If the bug occurs with booting the USB drive, file a bug report for Enterprise.
